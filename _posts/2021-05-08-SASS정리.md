# Sass와 SCSS 차이점
- {}(중괄호)와 ;(세미콜론)의 유무로 다르다
- Mixins('믹스인'은 재사용 가능한 기능을 만드는 방식)

## Sass
{% highlight html %}
.list
  width: 100px
  float: left
  li
    color: red
    background: url("./image.jpg")
    &:last-child
      margin-right: -10px
{% endhighlight %}

## SCSS:
{% highlight html %}
.list {
  width: 100px;
  float: left;
  li {
    color: red;
    background: url("./image.jpg");
    &:last-child {
      margin-right: -10px;
    }
  }
}
{% endhighlight %}

# 컴파일 방법
- Sass(SCSS)는 웹에서 직접 동작 불가능
- CSS로 변환후 실행해야함

## SassMeister 사이트
<https://www.sassmeister.com/>
{% highlight html %}
.container{
    width:100px;
    /* 중첩가능 */
    .item{
        width: 200px;
    }
}

### Sass의 경우 컴파일 되는 여러 줄 주석을 사용 시 * 을 사용하며 * 의 라인을 맞춰줘야 한다

 * 여러 줄
 * 주석 */

/* 컴파일되는

    * 주석 */
</code>
</pre>

### SCSS는 각 줄에 * 이 없어도 문제되지 않아 css와 호환이 쉽다

</code>
</pre>

# 데이터 종류(Data Types)
* Numbers   - 숫자                                       1, 10px, 2em ...
* String    - 문자                                       bold, relative, "dotum"
* Colors    - 색상
* Booleans  - 논리
* Nulls     - 아무것도 없음
* Lists     - 공백이나 ,로 구분된 값의 목록               (apple, orange, banana), apple orange
* Maps      - Lists와 유사하나 값이 key:value 형태       (apple:a, orange:o, banana:b)

# 데이터 종류의 특이사항
- Numbers: 숫자에 단위가 있거나 없음
- Strings: 문자에 따옴표가 있거나 없음
- Nulls : 속성값으로 mull이 사용되면 컴파일 되지 않는다.
- Lists: ()를 붙이거나 붙이지 않는다
- Maps: **()를 꼭 붙여야 합니다**
